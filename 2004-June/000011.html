I am not sure what you mean, but I'll describe the current situation and
let you guys decide.

The current Debian package of fetchmail is a patched version of
fetchmail, with the addition of a debian directory in the toplevel of
the source. Now, that development has picked up again, I'd like to
either merge or drop the Debian-specific patches into the trunk, so
that the Debian package only ends up as the trunk plus the debian
directory.

The reason I'd like to keep this in the same repository as fetchmail is
because it would make merging stuff from the Debian package to and from
the main fetchmail source easy. This will facilitate me updating the
Debian package, as well as me working on merging Debian-specific patches
into the trunk.

So, what do you guys think?
