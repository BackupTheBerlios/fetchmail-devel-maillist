Hello,
The warning cleanups have allowed me to notice bugs on some more exotic
architectures which were previously lost in the noise:

- in several places the *printf format string doesn't agree with the
  parameter types
- glibc doesn't export res_* on architectures with newer ABI, only
  __res_* and defines macros in <resolv.h>, thus configure would not
  detect res_* presence.

The attached patches fix these problems.
Thanks,
	Mirek
