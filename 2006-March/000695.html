Héctor García <hector at debian.org> writes:

> This patch would allow to specify the pid dir independent of the home
> dir.
> This would allow to have fetchmail-UIDL-cache file on /var/lib/fetchmail
> and pid file on /var/run/fetchmail
>
> This is needed when you run a global fetchmail with non-root user (as we
> do on Debian)
>
> So far this patch allows to specify pid dir with env var FETCHMAILPIDDIR

This patch lacks documentation (fetchmail.man), and is not needed.

You can use FETCHMAILHOME and then override the UIDL file location with
--idfile /var/lib/fetchmail/foo-uids.

Remember that the user running fetchmail needs write access
(create/rename files) to the directory.

I'll reconsider it for 6.4.0 (it's on my TODO),
but don't rely on it being applied.
