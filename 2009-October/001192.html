As of 6.3.10, the sleeping at/awakened at messages are
no longer logged unless in verbose mode. This gets rid
of a lot of noise, but in a typical non-verbose log
(mine, anyway) these were the only messages that
contained timestamps. 

Now, non-verbose logs do not contain any time markers,
making it very hard to correlate fetchmail log messages
with log messages from other components in an email
system. 

Fetchmail logs would be much more useful if at least
some common messages contained timestamps.

Personally, I don't care if log timestamps are on all
the time or are optional. I would always enable them.
Others might wish for the option to turn them off.

Thanks!
