Comment:
This is not a bug, but an operator error. Please:

1. update to a supported version of Fedora Core. I don't care about unsupported versions, and chances are your FC8 is vulnerable to various flaws.

2. read the manual page. Current versions of fetchmail support this:

   ESMTP AUTH
       fetchmail  also  supports  authentication  to  the  ESMTP server on the
       client side according to RFC 2554.  You  can  specify  a  name/password
       pair  to be used with the keywords 'esmtpname' and 'esmtppassword'; the
       former defaults to the username of the calling user.
