> I'm not really concerned as much about updates to an snprintf
> implementation; I wouldn't think there would need to be much updating.

What do other projects do with regard to snprintf or lack of it? How many
Unix systems lack snprintf these days?

According to the FreeBSD manpage,

     the snprintf() and vsnprintf()
     functions conform to ISO/IEC 9899:1999 (`ISO C99'').

Would it be reasonable simply to expect a decent O/S to have these functions
now? I don't know if SunOS 4.1 has snprintf or not, but if it doesn't and
you're still running it, you'll have worse problems than not being able to
compile fetchmail.

If there is a free-standing external snprintf library, then we could just
point people at it and tell them to install it first if it's needed.
