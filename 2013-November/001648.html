Joe,

As an update, I just polled my pop3.live.com accounts and figured that
--uidl works for me on that account.  Note that using --keep without
--uidl is a dangerous endeavor, meaning that if you use LAST or a
locally recorded message number, this setup is prone to mail loss or
skipping, should messages before the designated start number disappear
or should a downloaded message fail to be forwarded.

That comment above does not rule out my taking your patch, it may be
useful to re-setup things if the UID database should ever get damaged
(although I have not had reports of corruption of the .fetchids file in
a long time), we may however need to extend your patch so that if you
say "--startnum 1001", fetchmail could optionally record the first 1000
POP3 UIDs as seen.

Hope that passes for a comment for the nonce :-)

Thanks again!
