Hi Matthias,

http://lwn.net/Articles/192409/ tells us why it's useful for FLOSS to talk
with MAPI, and OpenChange are working on two different aspects: provide
interoperability with Exchange protocols and provide a transparent
replacement to Microsoft Exchange Server with native Exchange protocols
support and direct communication with Microsoft Outlook.

Fetchmail will be able to pull mail from Exchange, as well as push mail to
Exhange, if MAPI support is added. It seems that there are few software
under linux platform which support MAPI. One I know is Evolution, which
provides MAPI support with a plugin developed by OpenChange. So it will be
useful to add MAPI support to Fetchmail.

It's not decided whether I am the chosen one of this GSoC project, and
this will be decided before April 21, 2008. However, I'd like to contribute
to  this project even if I'm not the chosen one.
