fetchmail-7.0.0-alpha2+MAPI.tar.xz passes basic testing (IMAP, not MAPI) on Mac OS X 10.7.1 'Lion':

configure had no complaints I could spot
make reported a lot of SSL-related deprecation warnings, but was otherwise happy
gcc is i686-apple-darwin11-llvm-gcc-4.2 (GCC) 4.2.1 (Based on Apple Inc. build 5658) (LLVM build 2335.15.00)
installed into /usr/local
fetches via IMAP from Microsoft Exchange Server 2003 IMAP4rev1 server version 6.5.7638.1
with config settings:
set postmaster "jrepenning"
set bouncemail
set no spambounce
set properties ""
#set logfile /Users/jrepenning/Library/Logs/fetchmail.log
set syslog
# poll option idle gives quicker fetches, 
# but doesn't survive network changes
set daemon 300
poll exchange.collab.net with protocol IMAP
       user 'jrepenning' there with password 'XXXXX'
       is 'jrepenning' here
       stripcr ssl
       mda '/usr/bin/procmail -d %T'
       fetchall
