> Otherwise you might end up redownloading thousands of messages just
> because you changed your retrival programm.

Hi Daniel,

indeed this is not in fetchmail yet, I've placed it in TODO.txt.

For the nonce, you can try to convert the UIDs from your previous
program, the .fetchids format is line-oriented (i. e. one line per UID)

login at server ID

and all different logins and servers mixed in the same .fetchids file.
Fetchmail does not expect a certain order, and there's a space (ASCII
code 0x20 hex/040 octal/32 decimal) between server and ID.

This is going to change in a later version, but not in 6.3.X.
